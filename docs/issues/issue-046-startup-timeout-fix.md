# Issue #46: 起動タイムアウトの改善

**作成日**: 2026-02-04
**優先度**: P0
**ステータス**: Open
**関連Issue**: #43

---

## 概要

クラスタ起動時に、最初の起動でタイムアウトが発生する問題を修正します。

---

## 現象

### 安定性テスト結果

10回の起動・停止サイクルを実施した結果：

| 回数 | 結果 |
|------|------|
| 1 | ❌ タイムアウト |
| 2 | ❌ タイムアウト |
| 3-8 | ✅ 成功 |

- **成功率**: 75% (6/8)
- **タイムアウト率**: 25% (2/8)

### 原因

クラスタ起動に時間がかかり、120秒のタイムアウトを超えた可能性があります。Issue #45の修正でプロンプト準備完了の確認に最大60秒待つようになったため、起動時間がさらに長くなりました。

---

## 修正内容

### タイムアウト時間の延長

**ファイル**: `scripts/manual-stability-test.sh`

```diff
- # 起動完了を待つ（最大120秒）
- MAX_CHECK=24
+ # 起動完了を待つ（最大180秒）
+ MAX_CHECK=36
```

---

## 期待される効果

- 最初の起動でタイムアウトが発生しなくなる
- 成功率が75%から100%に向上する

---

## テスト計画

1. 修正実施後、再度10回の起動・停止サイクルを実施
2. すべての起動でタイムアウトが発生しないことを確認
3. 成功率が100%であることを確認
