# ドキュメント更新フロー

このドキュメントでは、ドキュメント更新の運用について詳しく説明します。

---

## 基本原則

⚠️ **重要**: 新機能追加・アーキテクチャ変更時は、必ずドキュメントも更新してください。

---

## ドキュメント更新が必要なケース

以下の場合はドキュメントの更新が必要です：

- [ ] **新機能の追加**: 新しい主要コンポーネントを追加した場合
- [ ] **アーキテクチャ変更**: システム構造に大きな変更を加えた場合
- [ ] **プロジェクト構成の変更**: 新しいディレクトリやファイルを追加した場合
- [ ] **開発環境の変更**: 依存パッケージやツールを変更した場合
- [ ] **Phase完了**: Phaseが完了した場合は関連ドキュメントを更新

---

## 更新が必要なドキュメント一覧

### 主要ドキュメント

| ドキュメント | 更新タイミング | 内容 |
|-------------|---------------|------|
| `docs/roadmap.md` | Phase完了時 | Phaseのステータス、完了日、完了条件 |
| `docs/architecture.md` | 新規コンポーネント追加時 | システム全体図、コンポーネント説明 |
| `docs/overview.md` | Phase完了時 | Phase構成表、ディレクトリ構成 |
| `docs/specs/directory-structure.md` | 新規ディレクトリ追加時 | ディレクトリ構造、ファイル説明 |

### 専門ドキュメント

| ドキュメント | 更新タイミング | 内容 |
|-------------|---------------|------|
| `docs/web-dashboard.md` | Webダッシュボード変更時 | 機能説明、使用方法 |
| `docs/api/web-api.md` | API変更時 | エンドポイント仕様 |

---

## Phase完了時の更新チェックリスト

Phaseが完了した場合は、以下のドキュメントを更新してください：

- [ ] `docs/roadmap.md`
  - [ ] Phaseのステータスを「✅ 完了」に変更
  - [ ] 完了日を記入
  - [ ] 完了条件のチェックをすべてオンにする
  - [ ] Issue履歴に追加

- [ ] `docs/architecture.md`
  - [ ] 新規コンポーネントの説明を追加
  - [ ] システム全体図を更新

- [ ] `docs/specs/directory-structure.md`
  - [ ] 新規ディレクトリを追加
  - [ ] 新規モジュールを追加

- [ ] `docs/overview.md`
  - [ ] Phase完了状態を反映
  - [ ] ディレクトリ構成を更新

---

## ドキュメント更新コマンド

```bash
# 1. ドキュメントを編集
vim docs/roadmap.md

# 2. 変更を確認
git diff docs/

# 3. コミット（コード変更と一緒に）
git add docs/
git commit -m "docs: Phase X完了に伴うドキュメント更新"
```

---

## 担当者

ドキュメント更新の責任者：

- **実装者**: ドキュメントの更新を実施
- **レビュワー**: ドキュメント更新の漏れを確認
